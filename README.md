# ✈️ Flight Price Monitor (Aviasales → Telegram)

Бот мониторит цены на авиабилеты через **официальный API Aviasales
(Travelpayouts)** и присылает в Telegram **пары билетов туда+обратно**
(Пермь → Москва → Пермь) с поездкой на 1–2 недели, которые стоят **заметно
дешевле средней цены такой пары**.

По умолчанию настроен на: **Пермь (PEE) ↔ Москва (MOW), июль–август 2026,
7–14 дней в Москве**.

Хостинг — **GitHub Actions** (бесплатно), запуск по расписанию каждые 4 часа.

---

## Как это работает

1. По расписанию (`.github/workflows/monitor.yml`) запускается `monitor.py`.
2. Скрипт тянет минимальные цены односторонних билетов в обе стороны
   (PEE→MOW и MOW→PEE) по каждой дате нужных месяцев.
3. Составляет все **пары** (вылет туда, вылет обратно) с зазором
   `MIN_STAY`..`MAX_STAY` дней. Цена пары = сумма двух билетов.
4. Считает «обычную» цену пары = медиана по всем парам.
5. Пары дешевле обычной на ≥ `DROP_THRESHOLD` (по умолч. 15%) шлёт в Telegram —
   с датами, временем вылета/прилёта по местному и отдельной ссылкой на каждый
   билет. В одно уведомление кладёт топ-`MAX_ALERTS` самых дешёвых.
6. Уже отправленные пары запоминаются в `data/alerts.json` и повторно не шлются,
   пока не появится цена **ещё ниже**. История «пульса» цен пишется в
   `data/history.json`. Оба файла коммитятся обратно в репо (раннер Actions
   эфемерный, поэтому состояние храним в git).

> ⚠️ API отдаёт **кешированные** цены (по реальным поискам пользователей),
> а не строго live, и «пара» — это два отдельных билета в одну сторону.
> Финальную цену проверяй уже по ссылкам на самом Aviasales.

## Настройка (10 минут)

### 1. Получить токен Aviasales / Travelpayouts
1. Зарегистрируйся на <https://www.travelpayouts.com> (партнёрка Aviasales, бесплатно).
2. В разделе разработчика создай **API-токен** (Data API / "Токен для API").

### 2. Создать Telegram-бота
1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → получи **bot token**.
2. Напиши своему новому боту любое сообщение (например `/start`).
3. Узнай свой **chat_id**: открой в браузере
   `https://api.telegram.org/bot<ТВОЙ_BOT_TOKEN>/getUpdates`
   и найди `"chat":{"id": ... }`. Это число — твой `TELEGRAM_CHAT_ID`.

### 3. Залить в GitHub и добавить секреты
```powershell
git init
git add .
git commit -m "init: flight price monitor"
git branch -M main
git remote add origin https://github.com/<ты>/aviasales-parser.git
git push -u origin main
```
Затем в репозитории: **Settings → Secrets and variables → Actions → New repository secret**
и добавь три секрета:
- `TRAVELPAYOUTS_TOKEN`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

### 4. Включить Actions
Вкладка **Actions** → разреши workflow → запусти вручную кнопкой
**Run workflow** (`workflow_dispatch`), чтобы проверить. Дальше пойдёт по cron.

> На первом же прогоне придут текущие лучшие пары. Дальше бот молчит, пока
> не появится пара ещё дешевле.

## Локальный запуск (для теста)

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:TRAVELPAYOUTS_TOKEN = "..."
$env:TELEGRAM_BOT_TOKEN  = "..."
$env:TELEGRAM_CHAT_ID    = "..."
python monitor.py
```
Если токены Telegram не заданы — сообщение просто печатается в консоль.

## Настройки (через переменные окружения)

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ORIGIN` | `PEE` | Аэропорт вылета (Пермь) |
| `DESTINATION` | `MOW` | Назначение (Москва, все аэропорты) |
| `MONTHS` | `2026-07,2026-08` | Месяцы мониторинга, `YYYY-MM` через запятую |
| `MIN_STAY` | `7` | Минимум дней в Москве (обратный билет не раньше) |
| `MAX_STAY` | `14` | Максимум дней в Москве |
| `CURRENCY` | `rub` | Валюта |
| `DROP_THRESHOLD` | `0.15` | На сколько ниже средней цены пары, чтобы алертить (0.15 = 15%) |
| `MAX_ALERTS` | `5` | Сколько самых дешёвых пар класть в одно уведомление |
| `MAX_PRICE` | `0` | Игнорировать билеты дороже (0 = выключено) |

Менять их можно в блоке `env:` внутри `.github/workflows/monitor.yml`.

## Частота проверок
В `monitor.yml`, строка `cron: "0 */4 * * *"` — каждые 4 часа.
Хочешь чаще — поставь `"0 */2 * * *"` (каждые 2ч) или `"0 * * * *"` (каждый час).
Cron в GitHub Actions всегда в **UTC**.
